## 一、重新定义优先级：投资敏感度矩阵

### 信号敏感度分级

```
超高敏感（股价±10%+，立即影响）:
├─ 融资公告（估值跳变）
├─ IPO/并购（流动性事件）
├─ 监管行动（FTC起诉、禁令）
├─ 重大产品发布（GPT-5、Claude 4）
├─ 关键人物离职（CTO、CEO）
└─ 财务数据（Form 8-K）

高敏感（股价±5%，24小时内影响）:
├─ 大客户合作/流失
├─ 竞争对手降价/升级
├─ 监管听证会/提案
├─ 安全事故/宕机
└─ 诉讼（专利战、劳动纠纷）

中敏感（股价±2%，1周内影响）:
├─ 产品小更新
├─ 行业报告（卖方研究）
├─ 会议演讲
└─ 社区讨论热度

低敏感（间接影响，1个月+）:
├─ 招聘信息（扩张信号）
├─ 员工满意度
├─ 专利申请
└─ 学术论文
```

**结论**: 招聘、员工满意度确实是**低频慢变量**，不应占用核心注意力。监管、融资、产品发布才是**高频快变量**。

## 三、去掉的低敏感源（ROI低）

### ❌ 招聘信息（LinkedIn）
**原因**:
- 滞后2-3个月（招聘→产出）
- 不影响短期股价
- 信噪比低（招聘≠增长）

**替代方案**: 如果需要验证管理层"高速增长"说法，仅针对性调研，不常规监控

---

### ❌ 员工满意度（Glassdoor）
**原因**:
- 滞后3-6个月（不满→离职→影响）
- 慢变量，不适合日内/周内交易
- 员工抱怨≠公司业绩（Netflix评分低但股价好）

**替代方案**: 重大负面事件（大规模裁员、CEO性丑闻）才关注，通过新闻捕捉

---

### ❌ 专利申请（USPTO）
**原因**:
- 超长滞后（申请→产品，18-36个月）
- 专利≠商业成功（IBM专利多但不赚钱）
- 适合长期研究，不适合交易信号

**替代方案**: 改为监控产品发布（直接商业化）

---

### ❌ 学术论文（arXiv全量）
**原因**:
- 绝大多数论文无商业价值
- 论文→产品路径不确定
- 每日300+篇AI论文，信息过载

**保留方案**: 仅保留arXiv Sanity精选（社区筛选后）

---

### ❌ 通用科技媒体（TechCrunch, The Verge等）
**原因**:
- 新闻滞后（二手报道）
- 信息密度低（1篇有价值/10篇）
- 被Tier 1媒体（Bloomberg, Reuters）覆盖

**保留方案**: 仅保留Tier 1独家报道

---

### ❌ Reddit社区讨论
**原因**:
- 散户情绪≠机构行为
- 噪音极高（meme股风险）
- 不影响专业投资决策

**替代方案**: 极端情况（舆情危机）才人工调研

---

### ❌ App Store评分/Google Trends
**原因**:
- 消费者产品指标，AI多为B2B
- ChatGPT除外，但其数据无法免费获取
- 滞后于财务数据

**替代方案**: 等财报披露（官方数据更准）

## 五、警报分级（3级，严格过滤）

### 🔴 P0级（立即响应，5分钟内）

**触发条件**:
```python
P0_TRIGGERS = {
    # 监管
    "FTC起诉": "lawsuit|investigation|antitrust|monopoly",
    "DOJ行动": "criminal|investigation|charges",
    "欧盟禁令": "ban|prohibited|fine",

    # 融资/并购
    "SEC 8-K (收购)": "Item 1.01",
    "SEC 8-K (财务)": "Item 2.02",
    "IPO定价": "S-1/A.*price range",

    # 产品
    "大厂重大发布": "GPT-5|Claude 4|Gemini 2|Llama 4",

    # 人事
    "CEO/CTO离职": "resign|step down|departure",

    # 市场
    "VIX爆表": "VIX > 35",
    "个股暴跌": "单日跌幅 > 8%",
}
```

**处理流程**:
1. Slack/钉钉立即推送（@all）
2. 手机短信（如配置）
3. 自动生成3句话摘要：
   - **发生了什么**
   - **为什么重要**（影响估值/市场份额/监管风险）
   - **建议行动**（买入/卖出/观望/调研）
